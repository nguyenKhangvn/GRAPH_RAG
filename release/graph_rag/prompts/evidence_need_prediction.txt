## Task
Analyze the user query and output a JSON object with these fields:

```json
{{
  "anchor_entity": "the main entity the user is asking about (or null if none)",
  "anchor_type": "node type of the anchor (or null)",
  "target_type": "node type of the answer entity (or null if asking about any type)",
  "required_relation_roles": ["list of semantic_role values needed"],
  "constraints": [
    {{"attribute": "attribute_name", "value": "constraint_value", "operator": "contains|equals|gte|lte"}}
  ],
  "confidence": 0.0-1.0
}}
```

## Rules
- Output ONLY valid JSON, no explanation text.
- Use semantic_role values exactly as listed above.
- If the query doesn't reference a specific entity, set anchor_entity to null.
- If the query is open-ended (e.g., "có gì hay ở Gia Lai?"), set target_type to null and include broad relation roles.
- confidence reflects how confident you are in this prediction (0.0-1.0).
